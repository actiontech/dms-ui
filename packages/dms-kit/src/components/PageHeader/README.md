# PageHeader 页面头部

提供统一的页面头部布局，支持标题显示、额外内容区域和固定定位。

## 何时使用

- 需要页面顶部的标题显示
- 需要在标题右侧显示额外操作按钮或内容
- 需要固定定位的页面头部

## API

| 参数 | 说明 | 类型 | 默认值 | 必填 |
| --- | --- | --- | --- | --- |
| title | 页面标题内容 | `ReactNode` | - | - |
| extra | 标题右侧的额外内容 | `ReactNode` | - | - |
| fixed | 是否使用固定定位 | `boolean` | `false` | - |
| className | 自定义类名 | `string` | - | - |
| style | 自定义样式 | `CSSProperties` | - | - |

## 组件特点

1. **统一样式规范** → 自动应用项目主题系统的样式
2. **灵活布局** → 标题居左，额外内容居右，支持任意 React 节点
3. **固定定位** → 支持固定在页面顶部，适合长页面场景
4. **响应式设计** → 自动适配不同屏幕尺寸

## 注意事项

1. 组件需要包裹在 `ConfigProvider` 中确保主题正常工作
2. `title` 支持任意 React 节点，建议使用简洁的文本
3. `extra` 适合放置操作按钮、链接等交互元素
4. 固定定位模式会影响页面布局，注意使用场景

## 最佳实践

1. **标题简洁**：保持页面标题简洁明了，避免冗长的描述
2. **操作合理**：在 extra 区域放置常用的操作按钮
3. **布局平衡**：保持标题和额外内容的视觉平衡
4. **固定定位**：在长页面中使用 `fixed` 提升用户体验
