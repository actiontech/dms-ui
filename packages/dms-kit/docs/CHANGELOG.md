### 文档规范优化

- **组件文档标准化**
  - 所有组件文档遵循统一结构：何时使用、代码演示、API、组件特点、注意事项、最佳实践
  - API 表格规范化：扩展属性与常用属性分开，继承的属性链接到 Ant Design 官方文档
  - 组件特点统一使用箭头格式，保持 3-8 条，突出核心价值
  - 注意事项和最佳实践精简到 5-7 条，聚焦关键要点
  
- **Demo 示例规范化**
  - 添加必要的注释说明，特别是复杂配置和特殊处理
  - 代码保持简洁，移除冗余配置
